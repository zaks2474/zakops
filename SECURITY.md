# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Do not open public issues for security vulnerabilities.**

Please report security vulnerabilities by emailing: security@zakops.io

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 48 hours and provide updates every 72 hours.

## Security Measures

- All API endpoints require authentication
- HITL approvals for sensitive operations
- Audit logging enabled by default
- Regular dependency updates
- OWASP Top 10 testing in CI pipeline
